# 1. Introduction

## 1) Summary

장비를 여러 대 운용하는 환경에서는 현재 어떤 장비가 정상이고, 어떤 장비에 이상이 있는지 빠르게 확인하는 것이 중요하다. 그러나 장비 상태를 개별적으로 확인하거나 점검 결과를 따로 기록하면 관리 시간이 길어지고, 과거에 어떤 문제가 반복되었는지도 파악하기 어렵다.

이러한 불편함을 줄이기 위해 이번 프로젝트에서는 **Device Status Monitoring and Inspection Support System**을 제안한다. 이 시스템은 장비 상태를 한 화면에서 확인하고, 주의 또는 이상 상태가 발생하면 User가 알림 영역에서 이를 확인할 수 있도록 한다. 또한 상태 데이터와 점검 결과를 이력으로 남기고, 등록 장비 수와 주의/이상 장비 수 같은 간단한 통계를 제공한다.

## 2) Features of project

Device Status Monitoring and Inspection Support System은 장비 등록, 상태 입력, 이상 알림, 점검 기록, 이력 조회, 통계 조회를 지원하는 시스템이다. User는 장비 ID, 장비명, 위치를 입력하여 장비를 등록할 수 있고, 각 장비의 온도, 진동, 전원 상태와 같은 상태 데이터를 입력할 수 있다. System은 입력된 상태 데이터를 기준으로 정상, 주의, 이상 상태를 구분하고, 주의 또는 이상 상태가 있는 장비를 알림 영역에 표시한다.

구현 단계에서는 Java Swing을 사용하여 하나의 메인 화면에서 장비 목록, 상태 입력, 알림, 이력, 통계를 확인할 수 있도록 한다. 실제 장비 연동과 데이터베이스는 구현 범위에서 제외하고, 자동 생성 상태 데이터와 프로그램 실행 중 메모리 저장 방식으로 핵심 기능을 시연한다.

## 3) Goals

이번 Analysis 보고서에서는 Conceptualization 단계에서 정의한 Use case list를 바탕으로 Use case diagram을 작성하고, 각 Use case의 동작 흐름을 구체적으로 분석한다. 또한 구현에 필요한 Class를 도출하고, 실제 프로그램 화면을 바탕으로 User Interface prototype을 제시한다. 이를 통해 시스템이 제공하는 기능과 각 기능의 동작 흐름을 명확히 이해할 수 있도록 한다.

# 2. Use case analysis

## 2.1. Use Case Diagram

그림 2-1은 Conceptualization 문서에서 작성한 System context diagram과 Use case list를 바탕으로 한 Use Case Diagram이다. 이 시스템의 Actor는 Device와 User이다. Device는 System에 상태 데이터를 제공하고, User는 장비 등록, 상태 입력, 점검 수행, 장비 목록 조회, 이력 조회, 통계 조회, 이상 알림 확인 기능을 사용한다.

상태 데이터가 입력되거나 제공되는 과정에서 장비 상태가 주의 또는 이상으로 판단될 경우 Alert notification이 추가로 수행될 수 있다. 따라서 **Alert notification**은 **Input status data**와 **Provide status data**에 대한 `extend` 관계로 표현하였다. 반면 모든 기능이 반드시 공통으로 수행해야 하는 기능은 없으므로 `include` 관계는 사용하지 않았다.

# 3. Domain analysis

**1) DeviceMonitorApp**  
시스템의 전체 화면을 구성하고 주요 기능을 실행하는 Class이다. 장비 등록, 상태 입력, 상태 자동 생성, 점검 수행, 이력 조회, 통계 갱신과 같은 기능은 이 Class를 통해 실행된다. 또한 User가 버튼을 누르거나 값을 입력했을 때 발생하는 이벤트를 처리한다.

**2) Device**  
System에 등록된 장비 정보를 저장하는 Class이다. 장비 ID, 장비명, 위치, 온도, 진동, 전원 상태, 현재 상태 등급, 최근 확인 시간과 같은 정보를 가진다. 장비 목록 조회와 상태 입력 기능에서 사용된다.

**3) HistoryRecord**  
상태 입력 또는 점검 수행으로 생성된 기록을 저장하는 Class이다. 기록 시간, 기록 유형, 장비 ID, 장비명, 상태값, 상태 등급, 메모와 같은 정보를 가진다. Query history data 기능에서 이 Class에 저장된 기록을 화면에 표시한다.

**4) DeviceStatus**  
장비의 상태 등급을 구분하는 Class이다. 상태는 정상, 주의, 이상으로 나누며, System은 상태 데이터를 기준으로 장비 상태를 판단한다. 장비 목록, 알림 영역, 통계 정보에서 장비 상태를 표시할 때 사용된다.

**5) RecordType**  
이력 데이터의 종류를 구분하는 Class이다. 상태 입력으로 만들어진 기록과 점검 수행으로 만들어진 기록을 구분한다. 이를 통해 History view screen에서 기록의 유형을 확인할 수 있다.

**6) DeviceTableModel**  
장비 목록을 JTable에 표시하기 위해 사용하는 Class이다. Device Class에 저장된 장비 ID, 장비명, 위치, 상태값, 상태 등급, 최근 확인 시간을 테이블 형태로 제공한다.

**7) HistoryTableModel**  
이력 데이터를 JTable에 표시하기 위해 사용하는 Class이다. HistoryRecord Class에 저장된 상태 기록과 점검 기록을 시간 순서로 보여준다.

<img src="/images/ui-register-device.png" alt="ui-register-device" width="900"/>

</div>

<div align="center">

[그림 4-2] Register device

</div>

<br>

그림 4-2는 User가 새로운 장비를 등록하는 화면이다. User는 장비 ID, 장비명, 위치를 입력한 뒤 장비 등록 버튼을 누른다. 입력값이 정상적으로 입력되면 System은 해당 장비를 장비 목록에 추가한다. 장비 ID가 비어 있거나 이미 등록된 장비 ID인 경우에는 장비가 등록되지 않는다.

**3) Input status data and execute inspection**

<div align="center">

<img src="/images/ui-input-status.png" alt="ui-input-status" width="900"/>

</div>

<div align="center">

[그림 4-3] Input status data and execute inspection

</div>

<br>

그림 4-3은 User가 장비의 상태 데이터를 입력하거나 점검을 수행하는 화면이다. User는 대상 장비를 선택하고 온도, 진동, 전원 상태 값을 입력할 수 있다. 상태 데이터 저장 버튼을 누르면 System은 입력된 값을 기준으로 장비 상태를 정상, 주의, 이상 중 하나로 판단하고 장비 목록과 이력 데이터에 반영한다. 또한 User는 점검 수행 버튼을 통해 선택한 장비의 점검 기록을 남길 수 있다.

**4) Alert notification**

<div align="center">

<img src="/images/ui-alert-inspection.png" alt="ui-alert-notification" width="900"/>

</div>

<div align="center">

[그림 4-4] Alert notification

</div>

<br>

그림 4-4는 이상 알림을 확인하는 화면이다. 장비 상태가 주의 또는 이상으로 판단되면 알림 영역에 해당 장비 정보가 표시된다. User는 알림 영역에서 점검이 필요한 장비명, 위치, 상태값을 확인할 수 있다.

**5) Query device list**

<div align="center">

<img src="/images/ui-device-list-view.png" alt="ui-device-list-view" width="900"/>

</div>

<div align="center">

[그림 4-5] Query device list

</div>

<br>

그림 4-5는 등록된 장비 목록을 조회하는 화면이다. User는 장비 ID, 장비명, 위치, 온도, 진동, 전원 상태, 현재 상태, 최근 확인 시간을 확인할 수 있다. 이를 통해 등록된 장비들의 현재 상태를 한눈에 파악할 수 있다.

**6) History view screen**

<div align="center">

<img src="/images/ui-history-view.png" alt="ui-history-view" width="900"/>

</div>

<div align="center">

[그림 4-6] History view screen

</div>

<br>

그림 4-6은 상태 기록과 점검 기록을 확인하는 화면이다. User는 각 기록의 시간, 유형, 장비명, 상태값, 상태 등급, 메모를 확인할 수 있다. 이를 통해 과거에 어떤 장비에서 어떤 상태가 발생했는지 확인할 수 있다.

**7) Statistics view screen**

<div align="center">

<img src="/images/ui-statistics-view.png" alt="ui-statistics-view" width="900"/>

</div>

<div align="center">

[그림 4-7] Statistics view screen

</div>

<br>

그림 4-7은 장비 상태에 대한 요약 통계를 확인하는 화면이다. User는 등록 장비 수, 정상 장비 수, 주의/이상 장비 수, 누적 기록 수를 확인할 수 있다. 이 정보는 전체 장비 현황을 빠르게 파악하는 데 사용된다.
