# 1. Introduction

이 문서는 Analysis 단계의 다음 문서인 Design 단계의 문서이다. Analysis 문서에서는 시스템이 제공해야 하는 기능과 Use case를 정리하였다. Design 문서에서는 그 내용을 바탕으로 시스템을 어떤 Class로 구성하고, 각 기능이 어떤 순서로 동작하며, 주요 상태가 어떻게 전이되는지 나타낸다.

장비 상태 모니터링 및 점검 지원 시스템은 Java Swing 기반의 단일 화면 프로그램으로 설계한다. 설계 관점에서는 Device가 상태 데이터를 제공하고, System이 이를 수신하여 장비 상태 판단, 이상 알림, 점검 기록, 장비 목록 조회, 이력 조회, 통계 조회 기능을 수행하는 구조를 기준으로 한다. 다만 구현 단계에서는 실제 하드웨어 통신과 데이터베이스 연동을 제외하고, 자동 생성 또는 테스트 상태 데이터와 프로그램 실행 중 메모리 저장 방식을 사용하여 핵심 기능을 시연한다.

이 문서에서는 Class diagram, Sequence diagram, State machine diagram을 통해 구현 단계에서 사용할 정적 구조, 기능별 상호작용, 상태 전이 흐름을 설명한다.

# 3. Sequence diagram

이 장의 Sequence diagram은 Conceptualization 단계에서 정의한 Use case를 기준으로 작성한다. Lifeline은 주요 설계 객체를 `instance : ClassName` 형식으로 표시하고, 메시지는 Class diagram에 정의된 operation 이름을 우선 사용한다. 객체 생성은 `<<create>>` 메시지로 표현하며, 반복적으로 행 데이터를 조회하는 부분은 loop frame을 사용한다. 조회 기능처럼 TableModel이 직접 데이터를 제공하는 흐름은 DeviceTableModel 또는 HistoryTableModel을 명시하고, 단순 화면 갱신은 `updateDashboard()`로 표현한다. Class diagram에는 화면 표시와 데이터 변환에 필요한 보조 operation도 포함되어 있으므로, Sequence diagram은 Use case 수행에 직접 필요한 주요 메시지 흐름을 중심으로 표현한다.

# 4. State machine diagram

## 4.1. System state machine diagram

<div align="center">

<img src="/images/state-system-diagram.png" alt="state-system-diagram" width="900"/>

</div>

<div align="center">

[그림 4-1] System state machine diagram

</div>

<br>

그림 4-1은 프로그램 실행 후 System의 주요 상태 변화를 나타낸다. System이 실행되면 Dashboard screen 상태로 진입하며, Dashboard screen에 진입할 때 entry / updateDashboard()를 통해 화면에 표시되는 장비 목록, 이력, 통계, 알림 정보를 갱신한다. User가 상태 데이터를 저장하거나 Device 상태 데이터를 수신하면 status data updated 이벤트가 발생하고, System은 Status processing 상태로 이동하여 상태 평가를 수행한다. 상태 평가 결과가 NORMAL이면 Dashboard screen으로 돌아가고, WARNING 또는 ABNORMAL이면 Alert notification 상태로 이동한다. User가 알림을 확인하면 다시 Dashboard screen으로 돌아온다. 또한 User는 queryHistoryData()를 통해 History view screen으로 이동하여 이력 데이터를 확인할 수 있으며, select Dashboard tab 이벤트로 Dashboard screen으로 돌아올 수 있다. Dashboard screen에서 close application 이벤트가 발생하면 프로그램은 종료 상태로 전이된다.

## 4.2. Device status state machine diagram

<div align="center">

<img src="/images/state-device-status-diagram.png" alt="state-device-status-diagram" width="900"/>

</div>

<div align="center">

[그림 4-2] Device status state machine diagram

</div>

<br>

그림 4-2는 Device의 상태 등급 변화를 나타낸다. Device의 초기 상태는 NORMAL이다. NORMAL, WARNING, ABNORMAL은 Device가 가질 수 있는 실제 상태 등급이며, Current device status 안에 함께 표현하였다. Current device status는 NORMAL, WARNING, ABNORMAL을 포함하는 composite state이므로, 상태 데이터가 갱신되면 현재 sub-state가 무엇이든 Evaluating status로 전이된다. Evaluating status에서는 do / evaluateStatus()를 수행하여 상태 등급을 판단한다. 평가 결과가 NORMAL이면 NORMAL 상태로, WARNING이면 WARNING 상태로, ABNORMAL이면 ABNORMAL 상태로 전이된다. executeInspection()은 Device의 상태 등급을 변경하는 기능이 아니라 현재 상태를 확인하고 점검 기록을 생성하는 기능이므로 상태 전이 조건에 포함하지 않는다.
